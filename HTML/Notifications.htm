CoreMonitor

Notifications API

Part of the CoreMonitor WebAPI is a Notifications interface which provides the ability to send notifications to the device through web requests.
Web requests are made using either HTTP GET or HTTP POST requests to the /notifications/show page and contain the following parameters
  • title - The title that will appear on the notification, should not be longer than 20 characters
  • text - The text that will appear on the notification. This may include line breaks but shouldn't be more than 5 lines long
  • displayperiod - The amount of time in milliseconds that the notification will be displayed to the user, or 0 to wait for the user to dismiss it
The following are examples of well structured requests:
  • http://localhost:56302/notifications/show?title=Test&text=This%20is%20a%20test%20notification&displayperiod=1000
    Shows a notification with the following: { Title = "Test", Text = "This is a test notification", DisplayPeriod = 1000 }
Calls to the /notifications/show page will return a text based representation of the status of the function call. This may include one of the following:
  • success
  • failed
